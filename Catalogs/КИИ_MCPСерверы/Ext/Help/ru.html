Справочник описывает подключения к удалённым серверам по протоколу MCP (Model Context Protocol) поверх HTTP: обмен идёт в формате JSON-RPC 2.0, используются методы tools/list (список инструментов) и tools/call (вызов инструмента). Настройки из элемента справочника использует общий модуль КИИ_КоннекторMCP.
Назначение в конфигурации. По каждому серверу можно загрузить каталог инструментов в подчинённый справочник КИИ_ИнструментыИИ: при обновлении выполняется запрос к MCP, имена и описания инструментов сохраняются и синхронизируются с ответом сервера (отсутствующие на сервере записи удаляются). Секрет для доступа к API хранится не в реквизитах объекта, а в безопасном хранилище (см. поле ключа на форме).
Основные реквизиты
Наименование — произвольное имя подключения для отображения в интерфейсе.
Адрес — узел сервера для создания HTTP-соединения (имя хоста или IP без пути к ресурсу, например api.example.com). Путь к endpoint задаётся отдельно.
Адрес запроса — путь HTTP-запроса на этом узле (например /mcp или полный путь от корня сайта в том виде, в каком его ожидает ваш шлюз MCP).
Использовать защищённое соединение — при включении используется HTTPS (SSL).
Тип подключения — способ разбора ответа сервера: обычный HTTP (одно JSON-RPC сообщение в теле ответа); Streamable HTTP (несколько JSON-сообщений в потоке, разделённых переводами строк); SSE (ответ в виде потока событий text/event-stream, из ответа извлекается JSON-RPC с нужным идентификатором). Выберите вариант, который поддерживает ваш MCP-шлюз.
Тип авторизации и ключ API — тип из перечисления КИИ типы авторизации: для x-api-key ключ передаётся в одноимённом заголовке, для остальных заполненных типов с непустым ключом в запрос добавляется заголовок Authorization: Bearer …. Значение ключа после записи элемента сохраняется в безопасном хранилище и не хранится в виде обычного реквизита справочника.
Заголовки (табличная часть) — дополнительные пары «ключ — значение» HTTP-заголовков (например служебные заголовки прокси или идентификатор клиента). Они добавляются к запросу вместе с заголовками авторизации и типом содержимого.
Имя поля текста — какое поле брать у элементов content с типом text при разборе результата tools/call: обычно text; если сервер отдаёт текст в поле content, укажите content. Если не заполнено, используется text.
Команды формы элемента. Обе команды — Обновить инструменты и Проверить подключение — после записи элемента выполняют одну и ту же операцию: запрос tools/list и синхронизацию подчинённого справочника КИИ инструменты ИИ. Разница в интерфейсе: после успешного Обновить инструменты на форме обновляется таблица предпросмотра инструментов; при Проверить подключение выводится только результат в поле Статус. Если элемент ещё не записан, команды недоступны или сообщают о необходимости сначала записать элемент.
При ошибках запросов к MCP имеет смысл смотреть журнал регистрации по событиям подсистемы с префиксом КИИ.MCP.
