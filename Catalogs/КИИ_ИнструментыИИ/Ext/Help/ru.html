Справочник хранит инструменты (tools), объявленные на конкретном MCP-сервере: это подчинённый справочник по отношению к MCP серверы (КИИ_MCPСерверы). Каждый элемент соответствует одному инструменту в смысле протокола MCP и используется при настройке экспертов ИИ и при выполнении вызовов tools/call через общий модуль КИИ_КоннекторMCP.
Откуда берутся данные. Состав элементов заполняется и обновляется при выполнении команд Обновить инструменты или Проверить подключение в форме соответствующего MCP-сервера: выполняется запрос tools/list, после чего для каждого инструмента из ответа создаётся или обновляется запись, а элементы, которых больше нет в ответе сервера, удаляются (полная синхронизация с сервером).
Реквизиты
Наименование — имя инструмента на стороне MCP (поле name в ответе tools/list). Внутри одного владельца-MCP-сервера оно служит ключом при синхронизации: при переименовании инструмента на сервере при следующем обновлении может быть создан новый элемент, а старый удалён.
Описание — текстовое описание для модели и пользователя (поле description в протоколе).
Параметры JSON — схема аргументов инструмента в формате JSON (как правило объект JSON Schema из поля inputSchema). Эти данные передаются в контур выбора и вызова инструментов у языковой модели (см. функцию КИИ_КоннекторИИ.ИнструментыЭксперта).
Связь с экспертами ИИ. В справочнике Эксперты ИИ на вкладке с инструментами для строк с MCP указываются сервер и ссылка на элемент этого справочника. По ним подставляются имя, описание и параметры при формировании списка инструментов для запроса к модели; при вызове инструмента выполняется обращение к нужному MCP-серверу с именем из наименования элемента.
Ручное изменение. Технически реквизиты можно править вручную, но при следующем обновлении с MCP содержимое снова приведётся к данным сервера. Имеет смысл полагаться на автоматическое обновление и не полагаться на ручные правки как на постоянные.
